> J'ai installé à deux reprise plearn (sur des systemes ubuntu), et  
> j'ai toujours pu compiler le programme (meme plearn_python). Par  
> contre si j'execute pytest run --all
> la plupart des tests echouent. J'ai ajouté de pytest sur une  
> installation sur un systeme ubuntu hardy (8.04).
>
> Quels seraient les pas a suivre/essayer pour faire reussir plus de  
> tests?


Prenez un test qui ne passe pas, par ex. PL_SortRowsVMatrix . Vous  
pouvez trouver le répertoire où le test est situé en faisant
pyest locate nom_du_test
Si vous allez dans ce répertoire, vous y trouverez normalement un  
sous-répertoire .pytest à l'intérieur duquel il y aura 2 répetoires:
   expected_results contient les résultats normalement attendus de  
l'exécution du test
   run_results contient les résultats obtenus avec votre version

Vous pouvez ainsi comparer les 2. Aussi il se peut qu'un fichier de  
diff indiquant les différences soit produit.

Il se peut que de petites différences numériques dans les tests  
soient dûes à des différences de versions de compilateur ou de  
microprocesseur, mais normalement on utilise notre version modifiée  
de diff qui tolère de petites imprécisions numériques, donc ça ne  
devrait pas être la cause du problème.
