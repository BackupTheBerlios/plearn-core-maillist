Bonjour à tous,

Une petite remarque d'intérêt public concernant l'interaction entre  
numpy et matplotlib/pylab:: comme vous le savez sûrement, matplotlib  
contient sa propre couche d'abstraction numérique qui est supposément  
compatible Numeric/numarray/numpy.  Matplotlib appelle cela numerix.

CEPENDANT, il y a certaines différences comportementales entre une  
array numerix qui wrappe numarray versus numpy (certaines méthodes  
qui étaient présentes avec numarray qui ne le sont plus sous numpy,  
lorsque vus à travers la lentille de matplotlib::numerix).  En  
particulier, pour éviter tout ennui, l'ordre des imports devient  
important.  Vous devez inclure matplotlib AVANT numpy, pour que les  
méthodes de ce dernier aient préséance.

Par exemple:

	from numpy.numarray import *
	from pylab import *
	a = arange(100.)
	b = reshape(a, (10,10))
	print b.min()         # La méthode min n'existe pas!

alors que

	from pylab import *
	from numpy.numarray import *
	a = arange(100.)
	b = reshape(a, (10,10))
	print b.min()         # La méthode existe!

Vaut mieux être avertis que de rencontrer un plantage bizarre...
