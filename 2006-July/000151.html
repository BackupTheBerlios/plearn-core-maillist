On Wed, Jul 19, 2006, Pascal Lamblin wrote:
> On Wed, Jul 19, 2006, Pascal Vincent wrote:
> > >> Je suis surpris que ce soit tellement plus long. Je suspecte
> > >> que c'est parce que le nouveau parseur parse le fichier au
> > >> complet. As-tu essayé, pour voir, d'utiliser l'ancien "parseur"
> > >> (rudimentaire mais efficace) pour extraire rapidement les
> > >> directives préprocesseur non commentées, et utiliser le nouveau
> > >> parseur pour "interpréter" juste le petit bout de code résultant?
>
> Juste pour clarifier : je supprime les commentaires avant de parser.
> Je vais voir ce que ça donne si je supprime aussi les lignes de C++.

J'ai fait ça : j'ai rajouté une regexp qui ne garde que les directives
préprocesseur que je sais parser facilement, et je parse seulement ça.
Les résultats sont impressionnants : le temps d'exécution est seulement
15 % de plus que le pymake de base.

> > Du point de vue du C c'est une ligne! ;-)   un newline précédé du'un \ 
> > ne devrait jamais être considéré comme le commencement d'une nouvelle 
> > ligne...
> > Il faut considérer que DEFINITION: fin-de-ligne === séquence de \n et/ou 
> > \r non précédée d'un \
> 
> Oui OK, il faut juste que je voie comment faire comprendre ça à re.

J'ai cédé à la facilité : je remplace d'abord tous les '\\\n' par ' '.

> À mon avis, si on veut un parseur « rudimentaire » qui soit capable de
> gérer tous les cas ci-dessus correctement (même sans le support des
> strings), le temps de parsing va être encore multiplié par 3 ou 4, et on
> approchera du point où il vaudrait mieux utiliser cpp lui-même, surtout
> si on cache le résultat (je n'exclus pas cette solution, d'ailleurs).

Je pense que la solution avec parseur est acceptable telle quelle (sauf
si 15 % de pertes de performances vous gênent), donc je vais laisser
tomber le caching, même si à mon avis la seule solution vraiment clean
serait d'utiliser cpp et de cacher le résultat. Je le ferai peut-être un
jour si j'ai vraiment rien à foutre.

Merci de votre attention, et n'hésitez pas si vous avez d'autres
commentaires.
