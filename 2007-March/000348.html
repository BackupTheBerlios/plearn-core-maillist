On 26 Mar 2007, Jaonary Rabarisoa wrote:
> > Il se peut aussi que la ligne suivante fonctionne :
> > ptester.setOption("statnames", "[ E[test1.E[binary_class_error]] ]");
> 
> Je confirme, cette ligne marche. Sinon, j'ai aussi noté qu'on peut
> construire les statnames avec l'option statmask du PTester. Par contre
> j'ai pas bien compri comment ce dernier s'utilise. Quelqu'un peut il
> m'éclaissier un peu les idées la dessus ?
> 

Ca te permet de composer des operations (le * recupere le resultat de
l'operation precedente).
Par ex. dans l'exemple donne par "plearn help PTester" :
   statnames = [ "NLL" "mse" ]
   statmask  = [ [ "E[*]" ] [ "test#1-2#.*" ] [ "E[*]" "STDERROR[*]" ] ]
il va partir de
    [ NLL mse ]
puis appliquer E[*] pour donner
    [ E[NLL] E[mse] ]
puis appliquer test#1-2#.* pour donner
    [ test1.E[NLL] test2.E[NLL] test1.E[mse] test2.E[mse] ]
puis appliquer E[*] STDERROR[*] pour donner le resultat final :
    [  E[test1.E[NLL]]  STDERROR[test1.E[NLL]]  E[test2.E[NLL]] 
       STDERROR[test2.E[NLL]]
       E[test1.E[mse]]  STDERROR[test1.E[mse]]  E[test2.E[mse]] 
       STDERROR[test2.E[mse]] ]
